# 주문 목록

## 관리자 메뉴 위치

`주문` > `주문 목록`

## 메뉴 안내 및 설정 방법

<figure><img src="../.gitbook/assets/Info Banner.png" alt=""><figcaption></figcaption></figure>

**① 에스크로 연동 실패 안내**

* 에스크로 연동에 실패한 주문이 있으면 상단에 안내 배너가 표시돼요.
* `주문 조회`를 누르면 연동 실패 주문만 모아 볼 수 있고, 주문 상세에서 연동을 다시 시도하거나 PG사 관리자에서 확인할 수 있어요.

<figure><img src="../.gitbook/assets/image (20).png" alt=""><figcaption></figcaption></figure>

**② 주문 상태별 보기**

* 주문 상태별 품목 수가 표시되며, 탭을 누르면 해당 상태의 주문만 조회돼요. (최근 1개월 기준)
* 상태: 전체 · 배송준비중 · 배송중 · 배송완료 · 취소요청 · 교환요청 · 반품요청

**③ 주문 검색**

* 검색 조건을 선택하고 검색어를 입력한 뒤 Enter를 누르세요.
* 검색 조건: 주문자명, 휴대폰번호, 주문번호, 품목별 주문번호, 송장번호, 주문자 ID, 이메일, 수령인명, 수령인 휴대폰번호, 관리 메모, 상품명 등
* `상세 검색`을 누르면 기간·주문 상태 등 여러 조건을 함께 지정할 수 있어요.

**④ 검색 결과 건수 · 페이지당 개수**

* 조회된 주문·품목 건수가 표시돼요.
* 한 페이지에 보여줄 주문 수를 10 / 30 / 50 / 100건 중에서 선택하세요. (기본 50건, 설정 자동 저장)

**⑤ 주문 리스트**

* 기본 조회 기준은 최근 1개월 주문이며, 주문일 최신순으로 정렬돼요.
* 구성: 주문일, 상품 정보, 처리 상태, 결제 상태, 총 결제 금액, 주문자
* 상품 정보를 누르면 `주문 상세` 창이 열리고, 주문자(회원)를 누르면 `회원 상세`로 이동해요.
* 교환으로 다시 배송되는 상품에는 `교환` 뱃지가 표시돼요.
* 총 결제 금액에는 환불·환불 대기 금액이 제외돼요.

{% hint style="warning" %}
참고

조회된 주문이 없을 때는 "검색 결과가 없습니다." 문구가 표시돼요. 개별 주문의 처리는 [주문 상세](/broken/pages/0b6ffad4cf22e40d8e0f1cc7dc2d74ef12b50eff)에서 진행해요.
{% endhint %}
